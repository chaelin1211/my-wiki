## 이력 요약
1. 무중단 운영 환경 유지
	1. Q 소스를 직접 업로드 해서 수동 배포하는 방식을 활용했다고 했는데 어떻게 무중단 운영이 유지된 것인가요?
	2. Q 배포 실패 시 어떻게 복구 하셨나요? 복구 시나리오가 있나요?
2. 실시간 데이터 대시보드 구축으로 운영 가시성 개선
	1. Q 실시간 대시보드는 어떤 스택으로 구축하셨나요? 실시간성을 어떻게 구현하셨는지 궁금해요.
	2. Q 알람 기능도 있었나요? 어떻게 설계하셨나요?
	3. Q 대시보드 구축 전에는 이슈를 어떻게 감지하셨나요?
	4. Q 어떤 지표를 대시보드에 담으셨고, 그 지표들을 왜 선택하셨나요?
	5. Q 대시보드 덕에 빠르게 잡아낸 사례를 하나 들려주세요.
		1. 어떤 지표에서 이상을 감지했고, 무엇을 보고 원인을 좁혀갔으며, 어떻게 해결하셨나요? (감지 > 원인 분석 > 해결)
3. ETL 관련
	1. Q ETL 대상이 된 데이터의 종류와 규모가 어떻게 됐나요? 소스 시스템은 몇 개였고, 타겟은 어디였는지 궁금해요.
	2. Q ETL 도구는 뭘 쓰셨나요? 
		1. 상용 도구(Informatica, DataStage)인지, 오픈소스(Airflow, NiFi)인지, 직접 구현(Python/SQL 스크립트)인지요.
	3. Q 소스 데이터에 오류가 있어서 ETL이 실패했을 때 대처 방법이 뭐였나요?
		1. → 전체 중단 vs. 부분 처리 vs. 스킵 후 로깅. 판단 기준이 있어야 시니어.
