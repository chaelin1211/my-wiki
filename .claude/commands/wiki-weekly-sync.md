주간보고를 읽고 프로젝트 status를 갱신하라.

## 1. 주간보고 읽기

현재 날짜 기준으로 `weekly/YYYY-MM.md` 파일을 읽어라.
가장 최근 주 섹션(`### [YYYYMMDD-YYYYMMDD]`)을 찾아라.

## 2. 프로젝트 매핑

주간보고의 항목을 아래 기준으로 매핑하라:

| 주간보고 분류 | 경로 |
|-------------|------|
| 공통 > SQL Agent | projects/dna-sql-agent, projects/dna-sql-agent-web |
| 공통 > 제안서 | → 아래 제안서 매핑 로직 참조 |
| KAC | projects/kacportal, projects/kac-idp-noti |
| 공통 > 면접 | 스킵 (common/interviews/ 에 별도 파일로 관리) |
| 공통 (그 외) | 매핑 불명확으로 표시 후 확인 요청 |

### 제안서 매핑 로직

"공통 > 제안서" 항목을 만나면 아래 순서로 처리하라:

1. `common/proposals/` 하위의 모든 `status.md` 파일을 읽어 `name` frontmatter 값을 수집한다.
2. 주간보고의 제안서 항목 텍스트와 `name` 값을 비교해 가장 유사한 것을 매핑한다 (부분 일치 허용).
3. **매핑된 경우** → 해당 `status.md`를 갱신한다.
4. **매핑 실패(신규 제안서)** → 아래 절차로 자동 생성한다:
   - 제안서 이름에서 폴더명을 생성한다: 한글/영문 키워드를 추출해 kebab-case로 변환 (예: `통합데이터플랫폼 및 AI·BIM센터` → `통합데이터플랫폼-ai-bim`)
   - `common/proposals/{폴더명}/status.md`를 아래 형식으로 생성한다:

```markdown
---
type: proposal-status
name: {주간보고에서 추출한 제안서 전체 이름}
updated: {오늘 날짜}
---

# {제안서 이름}

## 완료된 것

## 진행 중

## 다음 할 일

## 일정

| 항목 | 일자 |
|------|------|
| | |

## 메모
```

   - 생성 후 이번 주 항목을 반영한다.
   - 보고 시 "신규 생성"으로 표시한다.

## 3. 각 status.md 갱신

`/Users/jeonchaelin/Documents/obsidian-vaults/my-wiki/{경로}/status.md` 를 읽고 아래 규칙으로 갱신하라:

- **TODO 항목** → "다음 할 일" 섹션에 추가 (중복이면 스킵)

## 4. Git 커밋 & 푸시

갱신된 파일이 하나라도 있으면 아래를 실행하라:

1. 변경된 파일만 스테이징한다 (`git add {변경된 파일 경로들}`).
2. 커밋 메시지 형식: `sync: {YYYY-MM-DD} 주간보고 반영` (오늘 날짜 사용)
3. `git push`로 원격에 반영한다.
4. 변경된 파일이 없으면 커밋은 생략한다.
