직전 주의 ING·TODO 항목을 이번 주 주간보고 섹션으로 이관하라.

## 1. 현재 주간보고 파일 읽기

현재 날짜 기준으로 `weekly/YYYY-MM.md` 파일을 읽어라.

## 2. 직전 주 섹션 파싱

파일에서 `### [YYYYMMDD-YYYYMMDD]` 형식의 섹션을 모두 찾아 **마지막 섹션**을 직전 주로 판단하라.

직전 주 섹션에서 아래 항목을 추출한다:

- `**ING**` 블록의 모든 항목 (들여쓰기·계층 구조 그대로 유지)
- `**TODO**` 블록의 모든 항목 (들여쓰기·계층 구조 그대로 유지)
- `**DONE**` 블록은 추출하지 않는다

## 3. 이번 주 날짜 계산

직전 주 시작일(YYYYMMDD)에 7일을 더해 이번 주 시작일을 구하고, 시작일 +4일을 종료일로 설정한다.

**"N째 주" 계산 규칙**

이번 주 시작일(월요일)이 해당 월의 몇 번째 월요일인지를 기준으로 한다:

- 1번째 월요일 → 첫째 주
- 2번째 월요일 → 둘째 주
- 3번째 월요일 → 셋째 주
- 4번째 월요일 → 넷째 주
- 5번째 월요일 → 다섯째 주

**월 경계 처리**: 이번 주 시작일이 다음 달이면 `weekly/YYYY-MM.md` (다음 달 파일)를 대상으로 한다. 파일이 없으면 새로 생성한다.

## 4. 이번 주 섹션 추가

대상 파일에 이번 주 섹션이 이미 존재하면 → ING·TODO를 병합하고 중복은 스킵한다.

존재하지 않으면 → 파일 맨 끝에 아래 형식으로 추가한다:

```
### [YYYYMMDD-YYYYMMDD] MM월 N째 주

**DONE**

**ING**

{직전 주 ING 항목 — 들여쓰기·계층 구조 그대로}

**TODO**

{직전 주 TODO 항목 — 들여쓰기·계층 구조 그대로}

```

## 5. Git 커밋 & 푸시

변경된 파일을 스테이징하고 아래 형식으로 커밋한 뒤 푸시한다.

커밋 메시지: `sync: {YYYY-MM-DD} {MM}월 {N}째 주 주간보고 이관`
