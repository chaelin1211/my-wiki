미팅 파일을 읽고 해당 프로젝트 status.md를 갱신하라.

## 1. 미팅 파일 찾기

`$ARGUMENTS`가 있으면 해당 경로를 미팅 파일로 사용한다.
없으면 `projects/` 하위의 모든 `meetings/` 폴더에서 오늘 날짜(`YYYY-MM-DD`)로 시작하는 파일을 찾는다.

## 2. 미팅 파일 읽기

미팅 파일의 frontmatter에서 `project` 필드를 확인한다.
없으면 파일 경로(`projects/{project}/meetings/`)에서 프로젝트 이름을 추출한다.

미팅 파일에서 아래 섹션을 파싱한다:

- `## 결정사항` 또는 `## 결론` → DONE 항목으로 처리
- `## 액션아이템` 또는 `## TODO` → TODO 항목으로 처리
- `## 진행 중` 또는 `## ING` → ING 항목으로 처리

섹션이 명시되지 않은 경우, 내용을 분석해 결정사항/액션아이템/논의사항을 직접 추출한다.

## 3. 프로젝트 status.md 갱신

`/Users/jeonchaelin/Documents/obsidian-vaults/my-wiki/projects/{project}/status.md` 를 읽고 아래 규칙으로 갱신하라:

- **결정사항 / 결론** → "완료된 것" 섹션에 추가 (중복이면 스킵)
- **액션아이템 / TODO** → "다음 할 일" 섹션에 추가 (중복이면 스킵)
- **진행 중 항목** → "진행 중" 섹션에 추가 (중복이면 스킵)
- `updated` frontmatter를 오늘 날짜로 갱신

## 4. 미팅 파일에 링크 기록

갱신 후 해당 프로젝트 status.md의 `## 메모` 섹션에 아래 형식으로 링크를 추가한다 (중복이면 스킵):

```
- 미팅: [[{미팅 파일의 vault 상대 경로}]]
```

## 5. 보고

갱신한 프로젝트, 반영된 항목, 미팅 파일 경로를 표 형식으로 보여줘.
섹션 구조가 불명확해서 자동 추출한 항목은 별도 표시하고 확인을 요청해라.
