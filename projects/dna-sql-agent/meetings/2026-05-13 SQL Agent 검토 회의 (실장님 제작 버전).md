### 1. 벡터 검색 정확도 개선 — 예상 질문 생성 방식
- 현재 컬럼 별로 예상 질문을 생성 → 질문이 단편적, 후보가 너무 많아 목표 테이블 외 테이블이 많이 선정되는 문제
- **결론**: 컬럼 단위가 아닌 **관계(relation) 기준**으로 예상 질문 생성 필요
- 연관관계에 대한 질문/정보 및 코드 샘플 데이터도 감안해야 함

### 2. few-shot 활용 전략
- 현아 프로) few-shot에서 정확도 문제를 해결할 수 있지 않나?
- 실장님) few-shot의 본래 용도는 **응답 예시**에 가깝지만, document용 RAG 구축 대신 **비즈니스 로직/도메인 지식 제공**으로 활용하고자 함
- **결론**: few-shot = domain knowledge 제공 용도로 확정
