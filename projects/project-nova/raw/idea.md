# Mail2Issue — 클라이언트 메일 자동 이슈화 Skill

> 메일·첨부파일의 요청사항을 AI가 분해해 Git 이슈로 등록·추적까지 자동화

---

## 컨셉

클라이언트가 보낸 메일이 도착하면 AI가 본문과 첨부파일(PDF·이미지·엑셀·PPT)을 함께 읽고, 요청사항을 작업 단위로 쪼개 GitHub/GitLab 이슈로 자동 등록한다. 모호한 요청은 명확화 질문을 포함한 회신 초안을 만들고, 일주일 이상 미응답·재언급된 이슈는 누락 위험으로 플래깅한다.

**핵심 차별점**

- 첨부파일 내부의 숨은 요청까지 추출 (예: "PPT 8페이지 참고해주세요" → 실제 8페이지를 읽고 인용)
- 클라이언트별 톤·실제 소요시간 패턴 학습
- 사람 검수 단계를 거친 안전한 회신 (자동 발송 X, Draft까지만)

---

## 동작 흐름

```
[1. 수신] → [2. 추출] → [3. 구조화] → [4. 등록] → [5. 회신·추적]
```

| 단계           | 내용                                           |
| ------------ | -------------------------------------------- |
| **1. 수신**    | Gmail/Outlook에서 라벨·발신자 기준으로 메일 트리거           |
| **2. 추출**    | 본문 텍스트 + 첨부파일(PDF·이미지·엑셀·PPT) 멀티모달 파싱        |
| **3. 구조화**   | 요청 분해 · 유형 분류 · 우선순위 · 담당자 자동 추정             |
| **4. 등록**    | GitHub/GitLab 이슈 자동 생성 (라벨·마일스톤·Assignee 포함) |
| **5. 회신·추적** | 회신 초안 Draft 저장 + 누락·재언급 감지 알람                |

**구조화 스키마 (각 이슈에 자동 매핑)**

- 제목 (한 줄 요약)
- 상세 내용
- 유형 (버그 / 기능요청 / 디자인수정 / 문의 / 긴급)
- 우선순위 (메일 톤·마감일 단서로 추정)
- 예상 담당자 (키워드 매칭)
- 관련 파일 첨부 링크
- 원본 메일 링크

---

## 구현 방식

### 사내 LLM + n8n (권장)

- **장점**: 내부 메일 데이터가 외부로 나가지 않음, 보안·데이터 주권 완전 확보, 도메인 최적화 가능
- **단점**: 첨부파일(PDF·이미지) 멀티모달 지원 여부 사전 확인 필요

### Claude API + n8n

- **장점**: 긴 컨텍스트·PDF·이미지 직접 분석, 첨부 파싱 품질 최고, 빠른 구축
- **단점**: 내부 메일이 외부 API로 전송됨 → 보안 검토 필요

### ChatGPT (GPTs / Actions)

- **장점**: 진입장벽 낮음
- **단점**: 내부 메일 외부 전송 이슈 동일, 대용량 첨부 한계

> **시작 전략**: 사내 LLM으로 텍스트 본문 파싱부터 PoC → 첨부파일 멀티모달이 필요한 경우 내부망 내 별도 파싱 모듈 추가 검토

**기술 스택**

| 영역 | 도구 |
|---|---|
| 메일 연동 | 사내 메일 시스템 연동 (API 또는 IMAP) |
| AI | 사내 LLM (텍스트 분석·구조화) |
| 첨부파일 파싱 | 별도 파싱 모듈 검토 필요 (PDF·이미지·엑셀) |
| Git 연동 | GitHub REST API / GitLab API |
| 오케스트레이션 | n8n, 또는 Python + cron |
| 알림 | Slack, Notion DB 동기화 |

## 리스크 관리

- **자동 발송 금지** — 회신은 Draft까지만 생성, 사람 검수 후 발송
- **자동 Close 금지** — 이슈 종료는 담당자가 직접 확인 후 처리
- **민감정보 필터** — 신용카드·주민번호 등 자동 마스킹
- **롤백 로그** — AI가 생성한 이슈·라벨링은 모두 추적 가능한 로그로 보존

## 발표 포인트

1. **페인포인트** — "주니어 PM 1명이 메일 정리에만 주 6시간 쓰는 문제"
2. **Before/After 데모** — 어수선한 메일 스레드 → 깔끔한 이슈 보드 시연
3. **정량 효과** — 건당 18분 → 3분, 누락률 12% → 0.5%
4. **확장 로드맵** — v1은 메일→이슈, 나머지는 모듈식 확장
