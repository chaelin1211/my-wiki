# project-nova — 아키텍처 (PoC)

## 시스템 개요

> 사내 메일 시스템에서 클라이언트 메일을 수신하면, 사내 LLM이 본문·첨부파일을 파싱해 요청사항을 구조화하고 GitLab/GitHub 이슈로 자동 등록한다. 회신 초안은 Draft로만 저장하며 사람이 검수 후 발송한다.

## 데이터 흐름

```
[메일 수신]
    ↓ 라벨·발신자 필터 (n8n 트리거)
[본문 + 첨부파일 파싱]
    ↓ 사내 LLM (텍스트) + 파싱 모듈 (PDF·엑셀·PPT)
[요청 구조화]
    ↓ 유형 분류 / 우선순위 추정 / 담당자 매핑
[Git 이슈 생성]
    ↓ GitLab API / GitHub REST API
[회신 초안 Draft 저장 + 알람]
    ↓ Slack 또는 사내 메신저
[사람 검수 → 발송]
```

## 기술 스택 상세

| 카테고리 | 기술 | 버전 | 용도 |
|---------|------|------|------|
| AI | 사내 LLM | - | 텍스트 파싱·구조화·분류 |
| 오케스트레이션 | n8n | latest | 워크플로우 전체 관리 |
| 메일 연동 | 사내 메일 API / IMAP | - | 메일 트리거 (확인 필요) |
| 첨부파일 파싱 | 검토 중 (PyMuPDF 등) | - | PDF·엑셀·PPT → 텍스트 변환 |
| Git 연동 | GitLab API | v4 | 이슈 자동 생성 |
| 알림 | Slack / 사내 메신저 | - | 이슈 생성·누락 감지 알람 |

## 외부 의존성

| 서비스 | 용도 | 비용 | 대안 |
|--------|------|------|------|
| 사내 LLM | 텍스트 분석 | 내부 | Claude API (보안 검토 필요) |
| 사내 메일 시스템 | 메일 수신 트리거 | 내부 | IMAP 직접 접속 |
| GitLab | 이슈 등록 | 내부 | GitHub |
