# ADR-001: LLM 선택 — 사내 LLM 우선 사용

## 맥락

Mail2Issue는 클라이언트의 내부 메일을 처리한다. 메일에는 계약 내용·일정·개인정보 등 민감한 정보가 포함될 수 있어 외부 API 전송 여부가 핵심 판단 기준이다.

## 선택지

### 옵션 A: 사내 LLM
- **장점:** 내부 메일 데이터가 외부로 나가지 않음, 보안·데이터 주권 완전 확보
- **단점:** 멀티모달(PDF·이미지) 지원 여부 사전 확인 필요, 성능이 외부 LLM 대비 낮을 수 있음
- **비용/노력:** 내부 인프라 활용, 추가 비용 없음

### 옵션 B: Claude API
- **장점:** 긴 컨텍스트·멀티모달 지원, 첨부파일 파싱 품질 최고
- **단점:** 내부 메일이 외부 서버로 전송됨 → 보안 검토 및 승인 필요
- **비용/노력:** API 비용 발생

## 결정

**옵션 A (사내 LLM)를 선택한다.**

## 근거

내부 메일의 보안 민감도를 고려할 때 외부 API 전송은 리스크가 크다. PoC 단계에서는 사내 LLM으로 텍스트 파싱부터 검증하고, 멀티모달이 필요한 경우 내부망 내 별도 파싱 모듈을 추가하는 방향으로 확장한다.

## 결과

- 사내 LLM의 멀티모달 지원 여부 확인 후 첨부파일 파싱 전략 재결정 필요
- 향후 성능 이슈 발생 시 옵션 B로 전환 검토 (보안팀 승인 전제)
